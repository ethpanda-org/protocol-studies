# 核心开发 

> ：警告：本文是一个[存根](https://en.wikipedia.org/wiki/Wikipedia:Stub)，通过[贡献](/contributing.md) 和扩展它来帮助维基。

核心协议规范、实现和测试的工作通常称为核心开发。核心开发人员是客户端软件、其验证和用于构建协议的相关工具的贡献者，以及本 wiki 中涵盖的所有领域。

各个独立团队开发了许多[共识](/wiki/CL/cl-clients.md)和[执行](/wiki/EL/el-clients.md)层的实现。核心开发人员与从事研究、测试和其他基础设施工作的团队一起维护在以太坊上构建的每个人所使用的基础技术。

![Space Core Devs](../../images/space-core-devs.png)

[Hsiao-Wei Wang](https://github.com/hwwhww) 在她于 2023 年 4 月在 ETHGlobal 东京举行的演讲“[以太坊核心开发/研究员的旅程”](https://www.youtube.com/watch?v=0lBrd2_fPPU)_中分享了上图。该图形象征着空间站与各个团队为实现空间站而付出的努力之间的联系。

从事单独客户端工作的团队分布在不同的组织、公司或非营利组织中。它们以不同的语言实现相同的规范，并具有不同的功能和性能配置文件。 [客户端多样性](https://ethereum.org/developers/docs/nodes-and-clients/client-diversity) 是以太坊所遵循的基本原则。不同客户端的范围可以保证网络的稳定性和安全性，同时保持核心开发的去中心化和对所有人的开放。 

尽管不同的团队来自不同的组织，但他们都齐心协力，协作改进整个网络。进展是通过公共渠道的讨论取得的，主要是在每周的会议上进行。这些调用是公开的，并在[项目管理仓库](https://github.com/ethereum/pm) 中进行跟踪，包括 ACD - 执行和共识层调用、各种“分组讨论室”和工作组调用。除了通话之外，以太坊 R&D Discord 频道、Eth Magicians 论坛和 EthResearch 论坛也进行了讨论。
